# Snort IDS Custom Rule Detection

## Overview

This lab demonstrates the configuration and testing of a custom detection rule using **Snort**, an open-source Network Intrusion Detection System (IDS). The objective was to create a custom alert rule, monitor network traffic, and verify that Snort successfully detected suspicious activity generated during testing.

---

## Objective

The objective of this lab was to:

- Install and configure Snort.
- Understand the structure of Snort rules.
- Create a custom IDS detection rule.
- Generate network traffic to trigger the rule.
- Verify that Snort generated the expected alert.
- Analyze the detection results.

---

## Scenario

As a SOC analyst, you are responsible for monitoring network traffic and detecting suspicious activity. A custom Snort rule was created to detect an Nmap scan targeting an FTP service. Test traffic was generated, and Snort was used to verify that the rule successfully detected the activity.

---

## Activities Performed

- Verified the Snort installation.
- Reviewed the Snort configuration file.
- Created a custom detection rule in the local rules file.
- Loaded the custom rule into Snort.
- Started Snort in IDS mode.
- Generated test traffic using Nmap.
- Verified that Snort generated alerts for the detected activity.
- Reviewed alert output and confirmed successful detection.

## Example Commands

Start Snort:

```bash
sudo snort -A console -q -c /etc/snort/snort.conf -i eth0
```

Edit the local rules file:

```bash
sudo nano /etc/snort/rules/local.rules
```

Generate test traffic:

```bash
nmap -p 21 172.17.0.2
```

---

## Example Custom Rule

```text
alert tcp any any -> any 21 (msg:"FTP Port Scan Detected"; sid:1000001; rev:1;)
```

## Investigation Findings

The custom Snort rule successfully detected the test network activity generated by the Nmap scan.

Observed during testing:

- Snort loaded the custom rule successfully.
- Network traffic was inspected in real time.

- Alert details confirmed the rule matched the expected traffic.
