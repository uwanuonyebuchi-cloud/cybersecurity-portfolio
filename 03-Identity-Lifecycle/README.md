# IT Support Lab – Identity Lifecycle

## Overview

This lab demonstrates my understanding of the Identity Lifecycle in an IT support environment. It covers how IT administrators create, manage, secure, modify, and remove user accounts throughout an employee's time with an organization.

## Identity Lifecycle Stages

### 1. User Onboarding
- Receive a new employee request
- Verify employee information
- Assign department and job role
- Prepare user account

### 2. Account Creation
- Create the user account
- Assign a username
- Generate a temporary password
- Configure email account

### 3. Access Provisioning
- Assign appropriate permissions
- Add the user to security groups
- Configure shared drive access
- Grant application access based on job responsibilities

### 4. Security Configuration
- Require password change at first login
- Enable Multi-Factor Authentication (MFA)
- Apply password policies
- Verify account security settings

### 5. Ongoing Account Management
- Reset passwords when requested
- Unlock locked accounts
- Update user information
- Modify permissions after promotions or department changes
- Remove unnecessary access

### 6. Account Monitoring
- Review account activity
- Investigate suspicious login attempts
- Verify compliance with company security policies
- Audit user permissions

### 7. User Offboarding
- Disable the user account
- Remove access to company applications
- Remove group memberships
- Archive email and files if required
- Document account closure

## Lab Scenario

A new employee joins the company and requires access to company systems.

The IT Support technician:

1. Receives the onboarding request
2. Creates the user account
3. Assigns a username and temporary password
4. Configures the employee's email account
5. Adds the user to the correct security groups
6. Enables Multi-Factor Authentication (MFA)
7. Verifies access to company resources
8. Documents the account setup

Later, the employee changes departments, requiring updated permissions. Eventually, when the employee leaves the company, the account is disabled, access is removed, and the identity is securely deprovisioned.
