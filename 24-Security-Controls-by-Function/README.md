# Lab 24 – Security Controls by Function

## Overview

This lab demonstrates an understanding of security controls and how they are categorized based on their primary security function. Security controls help organizations reduce risk by preventing, detecting, correcting, deterring, compensating for, or recovering from security incidents.

---

## Objective

The objective of this lab was to identify common security controls and classify them according to their function within an organization's cybersecurity program.

---

## Security Control Categories

### Preventive Controls
Preventive controls reduce the likelihood of a security incident occurring.

Examples:

- Firewall
- Security Awareness Training
- Background Checks

---

### Detective Controls
Detective controls identify security incidents after they occur.

Examples:

- Review System Logs
- Security Audits
- CCTV Monitoring

---

### Corrective Controls
Corrective controls help eliminate or reduce the impact of security incidents.

Examples:

- Updating Security Policies
- Fire Extinguisher
- Replace Broken Locks

---

### Deterrent Controls
Deterrent controls discourage attackers from attempting unauthorized actions.

Examples:

- Warning Signs
- Surveillance Cameras
- Security Guards
- Login Banner Warnings

---

### Compensating Controls
Compensating controls provide alternative protection when primary controls cannot be implemented.

Examples:

- Manual Approval Processes
- Strong Password Policies
- Logging and Audit Trails

---

### Recovery Controls
Recovery controls restore business operations following a security incident.

Examples:

- Restore from Backup
- Disaster Recovery Plan
- Backup Power Generator
